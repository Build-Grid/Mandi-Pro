# 🤝 Contributing to Mandi-Pro

First off, thank you for considering contributing to **Mandi-Pro** 🚀
We appreciate your time and effort in improving this project.

---

## 📌 Contribution Guidelines

To maintain code quality and consistency, please follow the guidelines below.

---

## 🧭 Development Workflow

1. **Fork** the repository
2. **Clone** your fork locally

    ```bash
    git clone https://github.com/Build-Grid/Mandi-Pro.git
    cd Mandi-Pro
    ```

3. Create a new branch

    ```bash
    git checkout -b system/feature/your-feature-name
    ```

4. Make your changes
5. Commit your changes using **conventional commits**
6. Push to your fork
7. Open a **Pull Request**

---

## 🧠 Branch Naming Convention

Use meaningful branch names:

- `frontend/feature/add-login`
- `backend/bug/api-timeout`
- `database/refactor/database-queries`

---

## 📝 Commit Message Convention

We follow **Conventional Commits**:

```
[BACKEND]: add login endpoint
[FRONTEND]: resolve worker creation bug
[DATABASE]: optimize queries
[DOCUMENTATION]: update setup instructions
```

### Types:

- `feature` → New feature
- `bug` → Bug fix
- `refactor` → Code improvement
- `documentation` → Documentation

---

## 🧪 Code Quality & Testing

Before submitting a PR:

- ✅ Ensure the project builds successfully
- ✅ Run all tests (if available)
- ✅ Follow linting rules
- ✅ Add tests where applicable

---

## 🔍 Pull Request Guidelines

When opening a PR:

- Link the related issue
- Provide a clear description
- Keep PRs small and focused
- Ensure all CI checks pass ✅

---

## 🚫 What to Avoid

- ❌ Large, unrelated changes in one PR
- ❌ Breaking existing functionality
- ❌ Skipping code review process

---

## 💬 Need Help?

If you have questions:

- Open a discussion
- Create an issue
- Reach out to maintainers at `thebuildgrid@gmail.com`

---

## 🎉 Thank You

Your contributions help make **Mandi-Pro** better for everyone.
We appreciate your support ❤️
